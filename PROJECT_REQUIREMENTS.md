# Agentic AI Hackathon - Project Requirements

**Event:** IUT 12th ICT FEST - Agentic AI Hackathon
**Dates:** 10 AM, 24 July - 10 AM, 25 July, 2026 (24 hours)
**Location:** IUT Premises, Gazipur

---

## 1. Team Composition

- **Size:** 1-3 members per team
- **Eligibility:** Currently pursuing an undergraduate degree
- **Cross-university teams:** Allowed
- **Multi-teaming:** Prohibited (one team per participant)

---

## 2. Project Rules (CRITICAL)

- Create a **new GitHub repository on-site** immediately after the Opening Ceremony
- Project must be developed **entirely on the day** of the hackathon
- **No pre-written source code** may be brought into the venue
- Projects developed (even partially) **before** the event will be rejected
- Open-source libraries, frameworks, APIs, and SDKs **are allowed**
- All open-source tools **must be mentioned** in the project README
- Any unfair attempts = immediate disqualification

---

## 3. Technical Guidelines

### Allowed
- Any publicly available APIs, SDKs, or your own trained AI/ML models
- Pre-trained models via API (OpenAI, Hugging Face, Azure Cognitive Services, Google Cloud AI, Anthropic Claude API)
- Any programming language, web framework, or mobile toolkit
- Cloud services (AWS, Google Cloud, Azure, etc.)

### Not Allowed
- Pre-written source code brought into the venue
- Projects started before the hackathon

---

## 4. Facilities Provided

- Wi-Fi connectivity (backup mobile data strongly recommended)
- Meals and refreshments for overnight stay
- On-site security

### Bring Your Own
- Laptop(s)
- Chargers
- Power strips / multiplugs
- Mobile data backup
- External keyboard/mouse (optional)

---

## 5. Submission Requirements

| Deliverable | Details |
|-------------|---------|
| **GitHub Repository** | Public repo with all code written during the hackathon. README.md must include setup, run instructions, and tech stack |
| **Demonstration** | Working prototype demo presented to judges |

**Deadline:** Strict. Late submissions will not be accepted.

---

## 6. Evaluation Focus (Recommended Submission Strengths)

To maximize judging scores, ensure your project demonstrates:

1. **Working end-to-end prototype** — not just slides or mockups
2. **Clear problem statement** with real-world relevance
3. **Effective use of agentic AI** — autonomous decision-making, tool use, multi-step reasoning
4. **Polished UI/UX** — clean, intuitive interface
5. **Scalability and feasibility** — could this ship as a real product?
6. **Strong README and documentation** — judges read this
7. **Impressive live demo** — practice it beforehand (in your head)

---

## 7. Code of Conduct

- Respect and professionalism at all times
- Collaboration and knowledge sharing encouraged
- No harassment, plagiarism, or misrepresentation
- Violations = disqualification and forfeiture of prizes

---

## 8. Win Strategy Checklist

- [ ] Decide tech stack before arriving (don't waste time choosing)
- [ ] Create repo skeleton + README template within first 30 minutes
- [ ] Divide work: 1 person on backend/AI, 1 on frontend, 1 on integration/demo
- [ ] Build MVP by hour 12, polish from hour 12-22, demo prep hour 22-24
- [ ] Test API keys and rate limits before coding
- [ ] Prepare a 3-minute demo script
- [ ] Have a fallback plan if a key API fails
